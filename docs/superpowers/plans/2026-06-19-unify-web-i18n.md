# kr-web + jp-web 통합 (단일 web + i18n) Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** `apps/kr-web` + `apps/jp-web`를 단일 `apps/web` 앱으로 통합하고, locale을 런타임 Host 헤더로 판별하여 경량 타입 사전 기반 i18n으로 처리한다.

**Architecture:** 신규 내부 패키지 `@4-pillars/i18n`이 `ko`/`ja` 타입 사전과 `t(locale)`를 제공한다. `apps/web`은 kr-web을 기반으로 복제하되, root loader가 요청의 Host/`?lang`/env로 locale을 판별해 하위로 내려준다. 시장별로 크게 다른 결제만 locale 분기 라우트로 흡수한다. 인증(locale → `buildAuthUrl` 쿼리 → API)은 기존 흐름 그대로 유지된다.

**Tech Stack:** pnpm workspace, Turborepo, React Router 7.16 (framework mode, SSR), TypeScript 5.9, Vitest 2, Tailwind 4.

## Global Constraints

- 패키지 매니저: pnpm@10.30.3, 내부 패키지는 빌드 없이 TS 소스 직접 소비 (`exports` → `./src/index.ts`).
- 내부 패키지 소스의 상대 import는 `.js` 확장자를 붙인다 (기존 `@4-pillars/ui` 관례).
- 모든 패키지 `package.json`: `"type": "module"`, `"version": "0.0.0"`, `"private": true`.
- `tsconfig`는 `tsconfig.base.json`을 extends (`strict`, `verbatimModuleSyntax`, `moduleResolution: bundler`).
- locale 타입은 정확히 `type Locale = "ko" | "ja"`, 기본값(fallback)은 `"ko"`.
- React 버전: `^19.2.6`. React Router: `7.16.0` (정확히 고정, kr-web과 동일).
- 기존 `@4-pillars/ui`의 `buildAuthUrl`/`LoginModal` 동작과 테스트는 변경 없이 통과해야 한다.

`apps/web/app/lib/locale.ts`:
```ts
export type Locale = "ko" | "ja";

export const DEFAULT_LOCALE: Locale = "ko";

function asLocale(value?: string | null): Locale | null {
  return value === "ko" || value === "ja" ? value : null;
}

function localeFromHost(host?: string | null): Locale | null {
  if (!host) return null;
  if (host.startsWith("kr.")) return "ko";
  if (host.startsWith("jp.")) return "ja";
  return null;
}

export function resolveLocale(input: {
  host?: string | null;
  langParam?: string | null;
  envLocale?: string | null;
}): Locale {
  return (
    asLocale(input.langParam) ??
    asLocale(input.envLocale) ??
    localeFromHost(input.host) ??
    DEFAULT_LOCALE
  );
}

export function localeFromRequest(request: Request, envLocale?: string | null): Locale {
  const url = new URL(request.url);
  return resolveLocale({
    host: request.headers.get("host"),
    langParam: url.searchParams.get("lang"),
    envLocale,
  });
}
```
