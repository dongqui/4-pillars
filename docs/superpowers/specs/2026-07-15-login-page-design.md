# 로그인 페이지 UI 설계

- 작성일: 2026-07-15
- 참고 시안: `design/project/Saju Login.dc.html`
- 범위: 정적 로그인 UI와 재사용 가능한 표시 컴포넌트. 실제 OAuth 연동은 제외한다.

## 결정 사항

- `/login` 경로에 독립 로그인 페이지를 만든다.
- 소셜 버튼은 hover와 focus 상태만 제공하며 클릭 동작은 연결하지 않는다.
- 카카오의 `최근 로그인` 배지는 prop으로 제어하고 기본값은 표시로 한다.
- 기존 공용 `Button`에는 브랜드별 스타일을 추가하지 않는다.
- 제공자별 차이는 설정 객체로 모으고 공통 버튼 컴포넌트가 이를 렌더링한다.

## 컴포넌트 구조

### `LoginPage`

`/login/page.tsx`에서 하위 컴포넌트를 조립한다. 페이지 자체에는 브랜드별 스타일이나 반복 마크업을 두지 않는다.

### `LoginShell`

상단 브랜드, 홈으로 돌아가기 링크, 중앙 콘텐츠 영역과 반응형 여백을 담당한다. 돌아가기 링크는 `/`를 사용한다.

### `SocialLoginButton`

`provider`와 선택적인 `showLastLogin`을 입력받는다. 공통 높이, 모서리, 아이콘 위치, 타이포그래피, hover/focus 상태를 한 곳에서 관리한다.

지원 provider:

- `kakao`
- `line`
- `google`
- `apple`

문구, 배경색, 글자색, 테두리, SVG 아이콘은 provider 설정 객체에 둔다. 버튼은 실제 인증을 시작하지 않으며 기본 `type="button"`을 사용한다.

### `LoginNotice`

이용약관, 개인정보 처리방침, 보안 안내를 렌더링한다. 링크 경로는 각각 `/terms`, `/privacy`를 사용한다.

## 시각 및 반응형 규칙

- 본문 최대 너비는 380px이다.
- 소셜 버튼 높이는 52px, 모서리는 14px, 버튼 사이 간격은 10px이다.
- 데스크톱과 모바일 모두 화면 중앙에 콘텐츠를 배치한다.
- 작은 화면에서는 수평 여백을 20px 이상 유지한다.
- 전역 Pretendard 설정과 기존 Tailwind 색상 체계를 재사용한다.
- 키보드 사용자를 위해 각 버튼과 링크에 명확한 focus-visible 스타일을 제공한다.

## 데이터 흐름과 오류 처리

페이지는 서버에서 가져오는 데이터나 인증 상태를 사용하지 않는다. `showLastLogin` 기본값만 컴포넌트 내부에서 처리한다. 네트워크 요청이 없으므로 로딩과 오류 UI는 범위에서 제외한다.

## 검증

- TypeScript 타입 검사와 ESLint를 통과해야 한다.
- Next.js 프로덕션 빌드에서 `/login`이 정상 생성되어야 한다.
- 데스크톱과 모바일 폭에서 시안의 정렬, 간격, 버튼 상태를 수동 확인한다.
- 동일한 버튼 마크업이 provider별로 중복되지 않았는지 확인한다.

## 범위 제외

- OAuth 리다이렉트 및 callback
- 세션과 사용자 상태
- 마지막 로그인 provider 저장
- 약관 및 개인정보 페이지 구현
- 로그인 실패·취소 메시지
